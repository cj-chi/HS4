# 總結與報告檔命名慣例

為方便之後整理與 GitHub 更新，**總結檔與報告檔**建議採以下命名方式。

---

## 格式

- **日期_主旨**  
  - 日期：`YYYYMMDD`（西元年月日）  
  - 主旨：簡短說明內容，可含底線或半形連字號，副檔名依類型（`.md`、`.json` 等）

範例：
- `20250227_實驗總結_三向比對與非線性結論.md`
- `20250227_三向比對_舊新混誤差表.md`
- `20250227_三向比對_舊新混誤差表.json`

---

## 放置位置

- **總結、架構、對照類**：`docs/`（例：實驗總結、流程對照、命名慣例）
- **當次實驗產出、數值報告**：`Output/` 或 `output/`（例：誤差表、截圖、JSON 報告）
