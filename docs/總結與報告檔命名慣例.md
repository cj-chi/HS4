# 總結與報告檔命名慣例

為方便之後整理與 GitHub 更新，**總結檔與報告檔**建議採以下命名方式。

---

## 格式

- **日期_主旨**  
  - 日期：`YYYYMMDD`（西元年月日）  
  - 主旨：簡短說明內容，可含底線或半形連字號，副檔名依類型（`.md`、`.json` 等）

範例：
- `20250227_實驗總結_三向比對與非線性結論.md`
- `20250227_三向比對_舊新混誤差表.md`
- `20250227_三向比對_舊新混誤差表.json`
- `Output/20250227_123456_三向比對_執行時間.json`（每次 run 一檔，供之後規劃與瓶頸分析用；時間單位為 s）

---

## 放置位置

- **總結、架構、對照類**：`docs/`（例：實驗總結、流程對照、命名慣例）
- **當次實驗產出、數值報告**：`Output/` 或 `output/`（例：誤差表、截圖、JSON 報告）
- **執行時間報告**：`Output/YYYYMMDD_HHMMSS_三向比對_執行時間.json`（三向比對腳本每次執行後另存，內容含各步驟 duration_sec 與 total_sec，單位 s，供預估整次 run 時間與瓶頸分析。可另加 `inferred_breakdown` 紀錄推定拆解，例如：第一張截圖 57.28 s 推定為「啟動遊戲約 53 s、換人物與截圖約 3 s」。）
