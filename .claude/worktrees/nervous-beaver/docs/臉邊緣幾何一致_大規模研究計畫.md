# 臉邊緣幾何一致：大規模研究計畫

## 目標

讓人物卡的正面投影臉緣輪廓與原始 JPG 的 FACE_OVAL 有**幾何一致**的特徵。

---

## 核心概念：Contour Width Profile（輪廓寬度剖面）

定義一個**兩邊完全相同的幾何量 x**：

```
給定一條閉合 2D 輪廓（FACE_OVAL 或遊戲正面臉緣），
在正規化高度 h ∈ [0, 1]（0=下巴, 1=額頭）處，
x(h) = w(h) / face_h
其中 w(h) = 該高度上輪廓的水平跨距（最右 x − 最左 x）
```

- **Source（原圖）**：從 MediaPipe FACE_OVAL 36 點算出 profile
- **Game（遊戲截圖）**：同一張截圖跑 MediaPipe，從**同一 FACE_OVAL 36 點**算出 profile
- **兩邊用完全相同的定義與算法** → 幾何一致

---

## 為什麼不需要拆 mesh

清查文件說「需從 ShapeAnime 還原 2D 投影量」→ 那是「source 端精確定義 x」的理想做法。但我們有更務實的路：

**用遊戲本身當 oracle**：
1. 設定某個 Slider 值 → 產卡 → 遊戲載入 → 截正面圖
2. 對截圖跑 MediaPipe → 得到 FACE_OVAL 36 點
3. 用**與 source 完全相同的函數**算 contour width profile

這樣：
- **x 的定義**在兩邊完全一致（同一函數、同一 FACE_OVAL、同一算法）
- 不需要知道遊戲內部如何 mesh/骨骼/蒙皮
- 只需要遊戲能跑、能截圖、MediaPipe 能偵測臉

---

## 實作步驟

### Phase 1：實作 Contour Width Profile

1. 寫 `contour_width_profile(face_oval_xy, K=20)` 函數
   - 輸入：FACE_OVAL 36 點的 (x, y) 座標
   - 輸出：K 個 `(h, w(h)/face_h)` 對
2. 整合到 `extract_face_ratios.py`：`extract_ratios()` 額外回傳 `contour_profile`
3. 在現有 source 圖上測試

### Phase 2：掃描基礎設施

1. 擴充 `run_onedim_face.py`（或新建 `sweep_contour_sliders.py`）：
   - 對 13 個 2D 可見臉緣 Slider（0, 2, 4, 5, 6, 8, 9, 10, 11, 13, 15, 16, 18）
   - 逐一掃描：固定其餘 Slider，只動該 Slider，從 -100 到 200
   - 每個值：產卡 → 截圖 → MediaPipe → 算 contour_profile
   - 產出：`slider_i_value → profile` 的 JSON 表

### Phase 3：優化器

1. 給定 source profile，搜尋 Slider 組合使 `||profile(game) - profile(source)||` 最小
2. 可用現有 Optuna/一維搜尋框架
3. Loss = Σ (profile_source[k] - profile_game[k])² 或加權

### Phase 4：端到端驗證

1. Source JPG → MediaPipe → contour_profile(source)
2. 優化 → 最佳 Slider → 產卡 → 遊戲截圖 → contour_profile(game)
3. 比對兩條 profile 曲線，量化殘差

## Slider 分類（決定策略）

| 類別 | Index | 策略 |
|------|-------|------|
| **2D 可見、影響輪廓寬度** | 0, 4, 5, 10, 15, 18 | Phase 2 掃描 + Phase 3 寬度優化 |
| **2D 可見、影響輪廓高度** | 2, 6, 9, 11, 13, 16 | 同上，但 x 取「高度方向的特徵」 |
| **2D 可見、角度** | 8 | 從 profile 或下巴段斜率推 |
| **深度（永遠做不到）** | 1, 3, 7, 12, 14, 17 | from_card，不動 |

## 與現有系統的關係

- **不取代**現有 16 ratio → 16 slider 的五官對齊（眼、鼻、嘴）
- **新增**臉緣 Slider 的 profile-based 對齊
- 最終 59 個 shapeValueFace = 五官(from_landmarks, 現有) + 臉緣(from_contour_profile, 新) + 深度/其他(from_card)
